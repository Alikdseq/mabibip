# Проверка защиты регистрации и подтверждения email (МаБибип)

## Переменные окружения

| Переменная | Назначение |
|------------|------------|
| `RECAPTCHA_SITE_KEY` или `RECAPTCHA_PUBLIC_KEY` | Публичный ключ reCAPTCHA v3 (Google) |
| `RECAPTCHA_SECRET_KEY` или `RECAPTCHA_PRIVATE_KEY` | Секретный ключ (только сервер) |
| `RECAPTCHA_VERSION` | Оставьте `v3` (невидимая капча) |
| `RECAPTCHA_MIN_SCORE` | Порог score v3, по умолчанию `0.5` |
| `RECAPTCHA_SKIP` | `1` / `true` — отключить проверку (локальные тесты, CI) |
| `RATELIMIT_ENABLE` | `0` — отключить лимиты (по умолчанию включены) |
| `SITE_BASE_URL` | Абсолютный корень сайта без слэша в конце, например `https://example.com` — для ссылок в письмах |
| `EMAIL_*`, `DEFAULT_FROM_EMAIL` | SMTP для `send_mail` (см. `config/settings/base.py`) |
| `YOOKASSA_ENABLED` | `1` — включить оплату через ЮKassa (webhook + создание платежей для «безопасной сделки») |
| `YOOKASSA_WEBHOOK_SECRET` | Секрет для подписи вебхука (заголовок `X-Webhook-Signature`) |
| `YOOKASSA_SHOP_ID` | Shop ID ЮKassa (Basic Auth) |
| `YOOKASSA_SECRET_KEY` | Secret key ЮKassa (Basic Auth) |
| `DEAL_PAYMENT_TIMEOUT_MINUTES` | Авто-отмена неоплаченных сделок через N минут (по умолчанию 30) |
| `DEAL_AUTO_CONFIRM_DAYS` | Автоподтверждение получения через N дней после «Отправлено» (по умолчанию 7) |

- **`RECAPTCHA_USE_GOOGLE_TEST_KEYS=1`** в `.env` на любом окружении — явно включить тестовые ключи из ТЗ.

## Подтверждение email

1. Зарегистрируйтесь с указанием email.

2. В логах консольного бэкенда (`EMAIL_BACKEND` для разработки) или в почтовом ящике найдите письмо «Подтвердите email — МаБибип».

3. Перейдите по ссылке вида `/accounts/email/verify/<uidb64>/<token>/` — должно появиться сообщение об успехе и редирект на вход.

4. Для корректных абсолютных URL в письмах задайте `SITE_BASE_URL`; иначе используется `request.build_absolute_uri` в момент отправки.

## Лимит регистрации (429)

С одного IP не более **3 POST-запросов в час** на `/accounts/register/` и `/accounts/register/sto/`. При превышении показывается страница с кодом 429 и пояснением.

В тестах лимит отключён (`RATELIMIT_ENABLE=False` в `config/settings/tests.py`).

## Тестовые номера и IMask

Маска ввода: формат `+7 (___) ___-__-__`. Для ручных проверок используйте валидные российские мобильные номера (как в тестах: `+7999…` после нормализации).

## Логин тестового Gmail

Учётные данные тестового почтового ящика в репозиторий **не** помещаются. Заказчик передаёт их отдельно (для SMTP `EMAIL_HOST_USER` / `EMAIL_HOST_PASSWORD`).

## Админка

В карточке пользователя доступны поля **Email подтверждён** и **Токен подтверждения email** (для поддержки и сброса заявок).

## Где обязателен подтверждённый email

### Правила (последняя версия)

- **Водители**: чтобы **смотреть/раскрывать телефоны**, нужно **добавить email** и **подтвердить** его по ссылке из письма.
- **Водители** могут **создавать/редактировать объявления** без подтверждения email.
- **Бизнес-роли** (мастер/автосервис/автомагазин): после **одобрения администратором** доступ **полный**, даже если email не подтверждён.

### Технически: где стоит блокировка

Если в профиле указан email, но он **ещё не подтверждён** по ссылке из письма (и аккаунт не является одобренным бизнесом):

- недоступен **кабинет автомагазина** (`/shops/cabinet/…`) и **совместимый** маршрут товаров СТО;
- в шапке сайта показывается жёлтая полоса со ссылкой на страницу **«Подтвердите email»** (`/accounts/email/pending/`), откуда можно **запросить письмо повторно**.

Просмотр/раскрытие телефонов в объявлениях блокируется, если email **не добавлен** или **не подтверждён** (кроме одобренных бизнес-аккаунтов).

## «Безопасная сделка» по объявлениям (псевдо-эскроу, как Авито — MVP)

### Как работает (кратко)
- **Покупатель** нажимает «Купить безопасно» на странице объявления → создаётся сделка и платёж в ЮKassa.
- После `payment.succeeded` по вебхуку деньги фиксируются как **«на холде»** для продавца (внутренний кошелёк).
- **Продавец** отмечает «Отправил(а)» → ставится `auto_confirm_at`.
- **Покупатель** нажимает «Получил(а)» (или срабатывает авто‑подтверждение через `DEAL_AUTO_CONFIRM_DAYS`) → средства **релизятся** в «доступно» продавцу.
- **Вывод** средств — через **заявку** в кошельке и **ручное подтверждение** в ERP.

### Отмена и возвраты (MVP)
- Пока продавец **не отправил**, покупатель может нажать **«Отменить»** — если платёж уже прошёл, оформляется **refund** через API ЮKassa.
- Авто‑отмена неоплаченных сделок через `DEAL_PAYMENT_TIMEOUT_MINUTES`.
