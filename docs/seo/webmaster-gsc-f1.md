# Фаза F1: Яндекс.Вебмастер и Google Search Console

Цель: подтвердить владение продакшен-доменом и передать поисковикам карту сайта.

## Перед началом (код и конфиг)

1. На проде задан **`SITE_BASE_URL`** — полный URL вида `https://example.com` **без** завершающего слеша (см. `config/settings/base.py`). От него зависят абсолютные ссылки в sitemap и `robots.txt`.
2. Выполнен **`python manage.py migrate`** (в т.ч. приложение `sites`) и при необходимости **`python manage.py sync_site_domain`**, чтобы `django.contrib.sites` совпадал с боевым доменом.
3. Публично открыты (200, без базовой авторизации):
   - `https://<домен>/robots.txt` — в нём указана строка `Sitemap: https://<домен>/sitemap.xml`;
   - `https://<домен>/sitemap.xml` — индекс или набор секций Django (`static`, `services`, `brands`, `stations`), см. `config/urls.py` и `apps/core/sitemaps.py`.

Дополнительно проверьте редирект с HTTP на HTTPS и единый канонический хост (с/без `www`).

## Яндекс.Вебмастер

1. [Вебмастер](https://webmaster.yandex.ru/) → добавить сайт → указать `https://<домен>`.
2. Подтвердить права (удобные варианты: **HTML-файл** в `static` + раздача через веб-сервер, **DNS TXT**, **Meta-тег** в `<head>` через шаблон — по политике команды).
3. После подтверждения: **Индексирование → Файлы Sitemap** → добавить URL:

   `https://<домен>/sitemap.xml`

4. Дождаться статуса «Успешно обработан» или исправить ошибки (404, таймауты, неверный XML).

## Google Search Console

1. [Search Console](https://search.google.com/search-console) → добавить ресурс **с префиксом URL** `https://<домен>/` (или доменный ресурс при управлении DNS).
2. Подтвердить владение (аналогично: HTML-файл, DNS, Google Analytics / Tag Manager — что уже есть в проекте).
3. **Sitemaps** → добавить ту же ссылку:

   `https://<домен>/sitemap.xml`

4. Убедиться, что статус «Успешно»; при предупреждениях прочитать детали (исключённые URL, ошибки загрузки).

## Критерий готовности F1

- Оба сервиса показывают **подтверждённое** владение сайтом.
- В каждом добавлен **один** sitemap URL (корневой `sitemap.xml`); переобход подхватит секции, которые отдаёт Django.

## Частые проблемы

| Симптом | Что проверить |
|--------|----------------|
| В sitemap «чужой» домен или `http://` | `SITE_BASE_URL`, `sync_site_domain`, HTTPS за прокси |
| Пустые секции `services` / `stations` | В каталоге нет видимых СТО/категорий — ожидаемо на пустом стенде |
| robots не видит sitemap | `robots_txt` в `apps/core/views.py`, что `reverse('seo_sitemap')` доступен с публичного хоста |
