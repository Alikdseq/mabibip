## Чеклист (каждый раз)

### Яндекс.Вебмастер

- [ ] **Индексирование → Страницы в поиске** — нет аномального падения числа страниц при росте контента.
- [ ] **Индексирование → История обхода** — нет массовых ошибок сервера (5xx) или долгих таймаутов.
- [ ] **Диагностика** — открытые критичные проблемы (дубли, «закрыто в robots», мобильные ошибки) — завести задачу или закрыть как ложные срабатывания.
- [ ] **Файлы Sitemap** — последняя обработка без ошибок.

### Google Search Console

- [ ] **Страница «Обзор» / Performance** — базовая динамика показов/кликов (даже при низких объёмах на старте).
- [ ] **Индексирование → Страницы** — «Проиндексировано» vs «Не проиндексировано»; для выборочных URL «Почему страница не проиндексирована» — осознанные причины (noindex, дубль, «ещё не обошли»).
- [ ] **Индексирование → Файлы Sitemap** — успешная обработка.
- [ ] **Удобство для мобильных** / **Core Web Vitals** (если доступно) — нет новых критичных URL; детали согласовать с `lighthouse-cwv-f3.md`.

### Сайт снаружи

- [ ] Главная и каталог открываются с мобильного (реальный телефон или эмуляция).
- [ ] Случайная карточка СТО и один лендинг `/uslugi/.../` — 200, корректный canonical (см. фазу A в `plan.md`).

## Когда эскалировать в разработку

- Рост 5xx или массовый отказ в индексации после релиза.
- Изменение `robots.txt` / редиректов, после которого падает обход.
- Регрессия: sitemap отдаёт не тот домен или пустой индекс на проде с данными.
