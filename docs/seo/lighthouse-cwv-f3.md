# Фаза F3: Lighthouse и Core Web Vitals (главная и каталог)

Цель: зафиксировать **базовые** метрики удобства и скорости на ключевых публичных страницах (в первую очередь **mobile**) и не допускать необоснованных регрессий перед релизами.

## Страницы для замеров

| Страница | Типичный URL |
|----------|----------------|
| Главная | `https://<домен>/` |
| Каталог СТО | `https://<домен>/sto/` |

При необходимости добавьте типичный лендинг услуги `/uslugi/<slug>/` с нагрузкой ближе к продакшену (изображения, блоки).

## Способ 1: Chrome DevTools (локально)

1. Откройте URL в Chrome в режиме **инкогнито** (меньше расширений).
2. **F12** → вкладка **Lighthouse** → устройство **Mobile** → категории Performance, Accessibility (по желанию Best practices, SEO).
3. Сохраните PDF или скрин отчёта; выпишите **LCP**, **INP** (или legacy **FID**), **CLS** из секции метрик (или из раздела Performance).

## Способ 2: PageSpeed Insights (прод-URL)

1. [PageSpeed Insights](https://pagespeed.web.dev/) → вставьте публичный URL главной и каталога.
2. Смотрите блок **Core Web Vitals** для мобильных и при необходимости «лабораторные» данные Lighthouse.

## Способ 3: CLI (опционально)

При установленном Node.js:

```bash
npx lighthouse https://<домен>/ --only-categories=performance --preset=mobile --output=html --output-path=./lh-home.html
npx lighthouse https://<домен>/sto/ --only-categories=performance --preset=mobile --output=html --output-path=./lh-catalog.html
```

Отчёты HTML можно приложить к релизу или положить в артефакты CI (без коммита в git, если политика такая).

## Перед релизом (регрессии)

Минимум:

- Повторить быстрый замер **главной** и **каталога** (mobile) после значимых изменений в шаблонах, статике, шрифтах, изображениях героя, HTMX-фрагментах каталога.
- Если LCP/CLS заметно ухудшились — разобрать в DevTools (Coverage, Network, Layout shift regions) до мержа или отката.

Полный автоматический Lighthouse в CI на каждый PR — опционально (требует стенда и бюджета времени CI).

## Критерий готовности F3

- Есть **заполненная строка** базовой линии для `/` и `/sto/` на целевом окружении.
- Команда договорилась, что перед крупными фронт-релизами выполняется **короткая** проверка по этому документу.
