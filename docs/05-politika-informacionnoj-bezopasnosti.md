## 1. Общие положения

1.1. Настоящая Политика информационной безопасности (далее — **Политика ИБ**) устанавливает цели, принципы и базовый набор организационных и технических мер по защите информации, обрабатываемой в информационных системах _________________________________________________ (далее — **Организация**, **Оператор**) в связи с эксплуатацией сервиса **«ПроМастер»** (далее — **Сервис**).

1.2. Политика ИБ разработана с учётом требований:
- Федерального закона от 27.07.2006 № 149-ФЗ «Об информации, информационных технологиях и о защите информации»;
- Федерального закона от 27.07.2006 № 152-ФЗ «О персональных данных»;
- Постановления Правительства РФ от 01.11.2012 № 1119 (меры по защите ПДн при их обработке в ИСПДн);
- Приказа ФСТЭК России от 18.02.2013 № 21 (состав и содержание организационных и технических мер);
- иных применимых нормативных актов РФ.

1.3. Политика ИБ является **внутренним документом** Организации. Фрагменты, не содержащие коммерческой тайны и не раскрывающие уязвимостей, могут публиковаться на Сервисе в обезобщённом виде для информирования пользователей.

## 2. Цели информационной безопасности

2.1. Обеспечение **конфиденциальности**, **целостности** и **доступности** информации, обрабатываемой при функционировании Сервиса.

2.2. Предотвращение **несанкционированного доступа**, утечек, искажения и утраты данных, в том числе персональных данных.

2.3. Обеспечение **непрерывности** работы Сервиса в разумных пределах и возможности **восстановления** после инцидентов.

2.4. Соответствие требованиям законодательства и договорных обязательств перед пользователями и партнёрами.

---

## 3. Область применения

3.1. Политика ИБ распространяется на:
- **прикладное ПО** Сервиса (веб-приложение, API, фоновые воркеры, административная панель);
- **СУБД** (PostgreSQL и др.), **кэш** (Redis), **брокеры очередей**, **файловые хранилища** резервных копий;
- **серверную инфраструктуру** (виртуальные машины, контейнеры, сети, балансировщики);
- **рабочие места** сотрудников, имеющих доступ к администрированию и ПДн;
- **каналы связи** (HTTPS/TLS, VPN при использовании).

3.2. **Исключения** оформляются актом Организации с оценкой рисков.

---

## 4. Роли и ответственность

4.1. **Руководство** утверждает Политику ИБ, выделяет ресурсы, утверждает ответственных.

4.2. **Администраторы / DevOps** обеспечивают настройку средств защиты, обновления, мониторинг, резервное копирование.

4.3. **Разработчики** соблюдают практики безопасной разработки (Code Review, запрет небезопасных конструкций, работа с секретами через переменные окружения).

4.4. **Пользователи внутренних систем** несут персональную ответственность за сохранность учётных записей и соблюдение режима конфиденциальности.

## 5. Классификация информации (упрощённая)

| Категория | Примеры | Требования |
| :-- | :-- | :-- |
| **Особо значимая** | Учётные данные админ-панели, ключи API, пароли БД, приватные ключи TLS | Хранение только в секрет-хранилищах / .env, без коммита в репозиторий, МФА для админов |
| **Персональные данные** | Телефон, e-mail, данные об авто, переписка | Шифрование каналов, ограничение доступа, журналирование админ-действий, резервные копии |

| **Публичная** | Описание Сервиса на сайте | Контроль целостности публикаций |

## 6. Организационные меры

6.1. **Доступ к ПДн и продакшен-инфраструктуре** предоставляется по принципу **минимально необходимого** (need-to-know).

6.2. **Учётные записи** персональные; запрещена передача паролей. Для администраторов — **двухфакторная аутентификация** (TOTP / аналог).

6.3. **Соглашения о конфиденциальности** с сотрудниками и подрядчиками, имеющими доступ к информации.

6.4. **Инцидент-менеджмент:** фиксация фактов инцидентов, реагирование, уведомление уполномоченных органов и субъектов ПДн — **при наличии законных оснований**.

## 7. Технические меры (целевое состояние для Сервиса «ПроМастер»)

### 7.1. Защита каналов и сессий

- Использование **HTTPS/TLS** для всех публичных интерфейсов; настройка **HSTS** на уровне веб-сервера (Nginx) / приложения.
- Безопасные настройки cookie: **Secure**, **HttpOnly**, **SameSite** (по возможности).

### 7.2. Аутентификация и пароли

- Хэширование паролей алгоритмами класса **bcrypt** / **Argon2** (через механизмы фреймворка Django).
- Ограничение попыток входа (**django-axes** или аналог), **капча** на критичных формах.
- **Скрытый URL** административной панели (не стандартный `/admin`); **МФА** для учёток администраторов (**django-otp** или аналог).

### 7.3. Защита от распространённых атак на веб-уровне

- **CSRF**-защита для форм; для API — токены в заголовках по правилам фреймворка.
- Экранирование вывода в шаблонах; **запрет** необоснованного `|safe` для пользовательского контента.
- Использование **ORM**; **сырые SQL** — только по исключению с обязательным security review.
- **Rate limiting** / throttling (**django-ratelimit**, лимиты Nginx `limit_req_zone`, лимиты DRF).

### 7.4. Шифрование и защита данных at-rest

- Шифрование дисков инфраструктуры (**LUKS** / возможности облачного провайдера) — по политике хостинга.
- **Резервные копии** БД и критичных артефактов: регулярность _______________, хранение в защищённом хранилище; **шифрование бэкапов** перед выгрузкой во внешние облака.
- При необходимости — **шифрование отдельных полей** ПДн на уровне приложения (ключи в секрет-хранилище).

### 7.5. Сегментация и сеть

- Размещение БД и Redis в **приватной сети**, недоступной из интернета напрямую.
- Firewall / security groups: только необходимые порты.

### 7.6. Логирование и мониторинг

- Централизованные **логи** приложения и веб-сервера; **ретенция** согласно правовым и операционным требованиям.
- Мониторинг доступности и ошибок; интеграция с **Sentry** (или аналог) для отслеживания исключений **без утечки** ПДн в сообщениях об ошибках (маскирование).

### 7.7. Обновления и уязвимости

- Регулярное обновление ОС, зависимостей, образов Docker.
- Просмотр advisories (GitHub Dependabot / аналоги).

### 7.8. CI/CD

- Автоматические **тесты** и **линтеры** в конвейере (GitHub Actions / аналог).
- Запрет деплоя при провале критичных проверок (кроме аварийных процедур с фиксацией).
