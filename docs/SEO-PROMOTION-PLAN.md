# SEO, индексация и продвижение МаБибип — пошагово (Яндекс/Google)

Документ написан простыми словами и по шагам. В конце — ТЗ для внедрения в проект (технические задачи).

> Важно: «чтобы браузеры сами продвигали сайт» — это на практике означает: поисковики (Яндекс/Google) должны **правильно проиндексировать**, понять тематику и качество страниц, увидеть пользу и поведенческие факторы. Никакой кнопки «продвигай» нет — работает система технического SEO + контент + локальная выдача + доверие (ссылки/упоминания) + скорость.

---

## A) Что уже есть в проекте (хорошая база)

В проекте уже реализованы базовые вещи:
- **`robots.txt`**: отдаётся по URL `robots.txt` (см. `config/urls.py`, `apps/core/views.py`, `apps/core/seo.py`)
- **`sitemap.xml`**: отдаётся по URL `sitemap.xml` (см. `apps/core/sitemaps.py`)
- **canonical + meta description + OG/Twitter**: в `templates/base.html`, canonical собирается в `apps/core/seo.py` через `apps/core/context_processors.py`

Это означает: фундамент есть, дальше нужно **расширить SEO‑покрытие** (в том числе мелких страниц), добавить **структурированные данные**, улучшить **контент**, подключить **аналитику/вебмастер**, и убрать причины плохой индексации.

## B) Что нужно сделать руками (без кода) — по шагам

### B1) Подключить Яндекс Вебмастер и Google Search Console

1. Подтвердить права на домен (лучше через **DNS‑запись**, надёжнее).
2. В Яндекс Вебмастер:
   - добавить сайт
   - указать главное зеркало (с `www` или без — как у тебя в проде)
   - загрузить/указать `sitemap.xml`
   - проверить раздел «Индексирование → Страницы в поиске» и «Ошибки»
3. В Google Search Console:
   - добавить домен (Domain property)
   - отправить `sitemap.xml`
   - проверить Coverage/Pages, Mobile usability

**Цель:** чтобы роботы быстро увидели sitemap и начали обход.

### B2) Подключить аналитику (Метрика + GA4)

1. Создать счётчик **Яндекс.Метрики**.
2. Включить:
   - вебвизор (если допустимо по политике)
   - цели: регистрация, отправка заявки бизнеса, создание объявления, отправка сообщения в чат, клик «позвонить», «добавить в избранное»
3. Создать **Google Analytics 4** и добавить тег.

**Цель:** понимать какие страницы приносят трафик, что люди ищут, где уходят.

### B3) Подключить Яндекс Бизнес (и карты)

Если есть юридические данные/контакты проекта:
1. Создать карточку организации «МаБибип» в **Яндекс Бизнес**.
2. Заполнить:
   - описание, категория, телефоны/почта
   - сайт
   - регионы присутствия
3. Добавить везде единый NAP (Name/Address/Phone) — чтобы поисковик доверял.

**Цель:** улучшить локальную выдачу и доверие.

### B4) Подготовить контент‑план (то, что реально поднимет запросы)

Нужно покрыть запросы вида:
- «ремонт автомобиля + город»
- «диагностика двигателя + город»
- «автоэлектрик + город»
- «СТО + город»
- «аренда авто + город»
- «аренда спецтехники + город»
- «купить запчасти + город»

Что делать:
1. Сделать 20–50 страниц‑лендингов под услуги/разделы/марки/город (часть уже есть через `apps/stations/landing.py`).
2. На каждой странице:
   - уникальный заголовок \(H1\)
   - 1000–2500 символов полезного текста
   - блок FAQ (5–8 вопросов)
   - ссылки на каталог/карточки исполнителей

**Цель:** дать поисковику “контентные ответы”, не только список.

### B5) Внешние сигналы доверия (упоминания/ссылки)

Минимум:
- профили в соцсетях \(VK, Telegram\) с ссылкой на сайт
- публикации в местных группах (ссылка на “каталог мастеров”)
- партнёрские ссылки от 10–30 СТО/магазинов (внизу сайта или в соцсетях партнёров)

**Цель:** повысить авторитет домена.

## C) Техническое SEO (что должно быть на сайте)

### C1) robots + sitemap — проверить на проде

Проверить в браузере на прод‑домене:
- `https://ВАШДОМЕН/robots.txt`
- `https://ВАШДОМЕН/sitemap.xml`

В `robots.txt` должен быть `Sitemap: https://ВАШДОМЕН/sitemap.xml` и запреты на личные кабинеты/апи.

### C2) Настроить единый «канонический домен»

Нужно решить одно зеркало:
- либо без `www`
- либо с `www`

Дальше:
- 301‑редирект со второго варианта на основной
- одинаковая схема **https**

### C3) Скорость и Core Web Vitals

Что сделать:
- включить сжатие (gzip/br) на Nginx
- кеширование статики (long cache)
- картинки только WebP/оптимизация, lazy‑load
- минимизировать тяжёлые JS на публичных страницах

**Цель:** чтобы сайт был быстрым на телефоне.

### C4) Микроразметка Schema.org (сильный буст)

Добавить JSON‑LD:
- Для карточки СТО/мастера: `LocalBusiness`/`AutoRepair` (адрес, телефон, часы, рейтинг/отзывы)
- Для объявления: `Product` или `Offer` (цена/валюта/состояние/город)
- Для хлебных крошек: `BreadcrumbList`
- Для FAQ на лендингах: `FAQPage`

**Цель:** расширенные сниппеты в поиске.

### C5) Политика индексации “мелких страниц”

Правило:
- Индексируем только **полезные публичные** страницы.
- Не индексируем:
  - кабинет/аккаунты/техстраницы
  - API
  - страницы с “мусорными” параметрами

## D) ТЗ по проекту (что улучшить в коде)

Ниже список задач для внедрения в проект (после согласования).

### D1) Уточнить canonical для объявлений автомобилей

Сейчас canonical для `classifieds:ads_list` фиксируется только по `tab` (см. `apps/core/seo.py`).  
После добавления новых режимов «Продажа/Аренда авто/Аренда спецтехники» нужно решить:
- **индексировать ли** отдельные выдачи аренды как отдельные страницы?  
Если да — canonical должен включать `deal` (а не только `tab=car`).

### D2) Добавить JSON‑LD микроразметку

Где:
- `templates/stations/station_detail.html` (или фактический шаблон карточки станции) — LocalBusiness/AutoRepair
- `templates/classifieds/ad_detail.html` — Offer/Product
- `templates/base.html` или шаблоны страниц — BreadcrumbList
- лендинги услуг/разделов/марок — FAQPage

### D3) Сгенерировать/вывести человеко‑понятные SEO‑тексты на лендингах

Проверить, что на лендингах услуг/разделов/марок есть:
- уникальные `seo_og_title`/`seo_meta_description`
- H1 соответствует запросу
- текст + FAQ

### D4) Автоматические meta title/description для “маленьких страниц”

Сделать единый механизм:
- если страница публичная и нет `seo_og_title` — подставлять шаблонный заголовок
- если нет `seo_meta_description` — формировать из сущности (название, город, услуги)

### D5) Проверить индексацию и запреты

Проверить список страниц, которые должны быть закрыты:
- /cabinet/, /accounts/, /api/, /chat/ и т.д. (уже есть в `robots.txt`)
Дополнить при необходимости `X‑Robots‑Tag: noindex` на приватные разделы (если нужно).

### D6) Контроль дублей URL

Проверить редиректы:
- http → https
- www → non‑www (или наоборот)
- слэши на конце URL (в проекте canonical добавляет `/`)
